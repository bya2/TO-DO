1장. 클라우드의 역할

1.1 시스템 기반과 기존 인프라의 문제점

 - 프로그래밍만으로 해결될 수 없는 일들이 있음.

 - 예) [ 보안이 확보된 네트워크가 구축, 데이터 보존을 위한 장소 확보, 서버나 네트워크 장애 발생에 대한 조치에 대한 운용 ]

 - 시스템에 요구되는 요건: *기능 요건, *비기능 요건

 *기능 요건: 시스템 기능으로서 요구되는 요건. (시스템이나 소프트웨어로 무엇을 할 수 있는지를 정리한 것, 프로그래밍으로 해결하는 경우가 대다수)
 *비기능 요건: 기능 요건 이외의 요건. 시스템 성능 및 신뢰성/확장성/유지보수성/보안 등과 관련된 요건

 ***결론: 따라서 비기능 요건을 충족시켜야하고, 시스템 인프라에 대한 지식이 필요.


1.1.1 시스템 기반

 *시스템 기반: 작성한 애플리케이션을 사용자가 언제든 이용할 수 있도록 환경을 뒷받침하는 기술 요소.

 - 예) 미들웨어, 운영체제, 하드웨어/네트워크, 서버군

 - 시스템 기반은 '데이터 센터'라는 장소에서 구축되어 운용되고 있음.

 - 시스템 기반 환경을 운용하려면,
    = 서버군에 설치되어 있는 OS, 미들웨어, 개발된 애플리케이션의 버전을 적절하게 관리.
    = 하드웨어의 장애나 CPU 사용률, 디스크 공간 등 리소스의 이상 유무를 식별하고 이상이 있으면 즉각 대처할 수 있는 구조나 체제를 갖추어야함.

 - 업무 시스템으로 이용되고 있는 주요 서버:
    = 업무 시스템은 다양한 서버들과 연계되어 운영.
    = 서버 종류: [ *애플리케이션 서버, *HTTP 서버, *데이터베이스 서버, *메일 서버, *DNS 서버, *DHCP 서버, *그룹웨어 서버, *프록시 서버, *NAT 서버, * 감시 서버, *인증 서버 ]


1.1.2 데이터 센터 보유의 문제점

 - 비용적인 문제


1.2 클라우드 시스템

 *클라우드 시스템: 시스템을 보유하는 것이 아니라, 필요할 때만 시스템을 이용한다는 관점에서의, 시스템의 이용 형태
 *클라우드: 네트워크상에서 다양한 서비스를 필요에 따라서 이용하는 시스템 형태.

 - 클라우드는 시스템 구축에 필요한 네트워크/서버/스토리지/애플리케이션 등을 '서비스'로서 제공.

 - 기업의 시스템 구성: [ *온프레미스, *퍼블릭 클라우드, *프라이빗 클라우드 ]

 *온프레미스: 회사 내 자체 데이터 센터 보유. 시스템 구축에서 운용까지 직접 수행. 이와 같은 형태.

 *퍼블릭 클라우드: '인터넷을 통해' 불특정 다수에게 제공되는 클라우드 서비스. (제공하는 서비스: IaaS, PaaS, SaaS)

 *프라이빗 클라우드: 특정 기업들로 구성된 그룹에게만 제공되는 클라우드 서비스. (그룹에 속한 기업 내에 데이터 센터를 공동으로 보유하는 형태)

 - 프라이빗 클라우드는 이용자를 한정하므로 보안이 용이하며, 독자적인 기능이나 서비스를 추가하기 쉬움.


1.2.1 클라우드 서비스 체계

 *IaaS: 인프라를 제공하는 서비스. (서버, 운영체제, 네트워크, 데이터를 저장하는 영역 등)

 *PaaS: 플랫폼을 제공하는 서비스. (하드웨어 외 개발 환경 등의 미들웨어, +IaaS)

 *Saas: 소프트웨어를 제공하는 서비스. (완성된 기능, 애플리케이션, 만들어진 시스템, +PaaS)

 *Daas: 클라이언트 단말 기능을 제공.


1.2.2 클라우드의 단점

 - 시스템 가용성은 클라우드 벤더가 보장.
 - 데이터의 정확한 보장 장소는 클라우드 벤더가 결정.
 - 구동되어야하는 시스템을 구축/이행할 필요가 있는 경우 클라우드 벤더가 반영.

 = 매우 높은 가용성이 요구되는 경우, 운용 불가

 = 물리적 보관 장소를 명확히 해야하는 데이터는 클라우드 상에 보관 불가

 = 범용적이지 않은 디바이스나 특수한 플랫폼에서 구동되어야 하는 시스템을 구축/이행할 필요가 있는 경우, 클라우드 벤더가 반영하지 못하면 이용 불가)

 = 전체적으로 시스템에 대한 투자 비용이 높아지는 경우, 온프레미스 형태로 구축하는 것이 경제적으로 좋음.
