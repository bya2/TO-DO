2.1.1 컴퓨팅

 - 아마존 EC2: 가상 서버(인스턴스) 기능.

 - 아마존 EC2 컨테이너 서비스: 도커를 운용하는 서비스.

 - 아마존 EC2 컨테이너 레지스트리: 도커 이미지를 보관하고 공유하는 서비스.

 - AWS Elastic Beanstalk: 앱을 AWS에 자동으로 배포.

 - AWS Lambda: 클라이언트의 요청 발생 시점에 임의의 프로그램을 구동시키는 이벤트 드리븐 형태의 서비스.

 - Auto Scaling: 사전에 정해진 조건에 의해서 EC2 인스턴스를 자동적으로 증감시키는 서비스.

 - Elastic 로드 밸런싱: 트래픽에 따라 복수 개의 EC2 인스턴스로 부하를 분산시키는 서비스.


2.1.2 스토리지 & 콘텐츠 전송 관련 서비스

 - 아마존 S3: 다중화된 데이터 스토리지 서비스. (업무 시스템에서의 파일 서버 같은 것)

 - 아마존 CloudFront: 전 세계에 콘텐츠를 전송하기 위한 네트워크 서비스.

 - 아마존 EBS: 아마존 EC2의 데이터를 보관하는 스토리지 서비스. (EC2의 하드디스크, SSD와 같은 역할, 스냅샷 백업 가능)

 - 아마존 Elastic 파일 시스템: EC2 공유 파일 저장 서비스. (자동으로 용량 스케일링)

 - 아마존 Glacier: 백업이나 아카이브 용도의 스토리지 서비스. (저렴함)

 - AWS Import/Export Snowball: 페타 바이트 급의 대용량 데이터 전송 서비스.

 - AWS 스토리지 게이트웨이: 온프레미스 환경과 AWS를 접속하는 스토리지 게이트웨이.


2.1.3 데이터베이스 관련 서비스

 - 아마존 RDB: 관계형 데이터베이스를 구축/운용하는 서비스.

 - AWS 데이터베이스 마이그레이션 서비스: 최소한의 정지 시간에 데이터베이스를 이행할 수 있는 서비스. (최소한의 정지 시간?)

 - 아마존 DynamoDB: NoSQL 데이터베이스 서비스를 구축/운용하는 서비스.

 - 아마존 ElastiCache: 클라우드에서 메모리의 캐시 관리가 가능한 서비스.

 - 아마존 RedShift: 빅데이터를 위한 데이터 하우스. (페타 바이트 규모)


2.1.4 네트워크 관련 서비스

 - 아마존 VPC: AWS 내에서 사설 네트워크를 구축하기 위한 서비스.

 - 아마존 Direct Connect: 온프레미스 환경의 네트워크와 AWS의 VPC 네트워크를 직접 접속하기 위한 전용선 서비스.

 - 아마존 Route 53: 도메인 네임과 IP 주소를 매칭시키는 DNS 시스템을 구축하기 위한 서비스.

2.2 AWS 클라우드 디자인 패턴

 *디자인 패턴: "A를 할 때 B하게 하는 것이 좋다"와 같은 노하우.

 *AWS 클라우드 디자인 패턴: 인프라를 구축할 때의 노하우.

 - 패턴: [*Multi-Server 패턴, *Scale Up 패턴, *DB Replication 패턴, *Functional Firewall 패턴 ]

    = Multi-Server 패턴(서버의 다양화): 복수 개의 인스턴스를 두고 ELB를 통해 로드 밸런싱. (인스턴스 장애로 인한 전체 시스템의 정지를 방지. 이는 시스템 가용성을 향상 시킴.)

    = Scale Up 패턴(서버의 확장/축소): 자동으로 서버를 스케일링. (기회 손실을 줄일 수 있음)

    = DB Replication 패턴(데이터베이스 복제): 가용 영역(다른 물리적 공간)을 따로 배치하고 그 안에 각각 마스터 데이터베이스와 슬레이브 데이터베이스를 둠.

    = Functional Firewall 패턴(계층별 접속): 서버의 역할에 따라서 접속을 제한하는 구성. (인스턴스에 보안 그룹 설정하고 필요 통신만 허가) (서버 1대마다 설정하면 설정 오류가 발생하기 쉽고, 불법적인 접근의 원인이 되므로, 서버의 계층마다 논리적인 그룹을 만들고 그룹 단위로 보안 그룹을 설정)


2.3 AWS 데이터 센터

 *리전(Region): 데이터 센터군이 설치되고 있는 지역.
 *가용 영역(AZ, Availability Zone): 물리적으로 격리된 장소. (네트워크 뿐만 아니라 완전히 다른 환경) (동일 가용 영역 안의 통신은 개인 IP 주소로, 접속 비용이 들지 않음) (동일 리전 내 가용 영역이 분리되었으면 인터넷을 경유하지 않지만 글로벌 IP 주소를 사용하여 접속하므로 통신 비용이 듬.)
