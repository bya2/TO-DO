네트워크 주소:
 - MAC 주소, IP 주소

NIC(네트워크 인터페이스 컨트롤러):
 - 네트워크 내에서 통신을 담당하는 하드웨어.
 - 유선 LAN의 경우, 네트워크 어댑터 등.
 - 무선 LAN의 경우, 와이파이 어댑터 등.

MAC 주소:
 - 물리적인 주소
 - 이더넷 주소

 - 네트워크 인터페이스 컨트롤러에 물리적으로 배정된 48비트의 주소. (16진수 표기, 2바이트씩 구분)
 - 앞 24비트: 네트워크 부품 업체 식별.
 - 뒤 24비트: 업체들이 중복되지 않도록 할당.(?)

 - 물리적인 네트워크 부품마다 고유하게 할당된 것. (원칙적으로 이용자가 임의로 변경 불가)

 - OSI 2 계층인 데이터 링크 계층에서 사용.

IP 주소:
 - 컴퓨터나 네트워크 기기에 할당된 식별 번호.

 - MAC 주소와는 다르게 네트워크 관리자가 서버나 기기의 NIC에 임의의 값을 할당할 수 있음.

 - IPv4, IPv6

IPv4:
 - 32비트가 네트워크 주소와 호스트 주소로 구성.

 - 네트워크 주소: 동일 네트워크에서 같은 값이 할당.
 
 - 호스트 주소: 네트워크 안에서 노드가 갖는 네트워크 인터페이스마다 할당. (같은 네트워크 내에서는 중복되는 IP주소를 할당할 수 없음)

 - 네트워크 주소와 호스트 주소 구별: 서브넷 마스크 표기, CIDR 표기, 

서브넷 마스크:
 - 1로 표시: 네트워크 주소
 - 0으로 표시: 호스트 주소
 - 8bit마다 10진수로 표시.
 - ex) 255.255.0.0

CIDR(Classless Inter-Domain Routing):
 - 네트워크 주소 부문을 몇 비트 사용할 것인지 IP 주소 뒤에 추가.
 - ex) 192.168.0.1 / 16

 - 네트워크 주소: 호스트 주소 부문이 모두 0
 - 브로드캐스트 주소: 호스트 주소 부문인 모두 1
