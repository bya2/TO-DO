네트워크 프로토콜:
 - 상호 간에 어느 정도의 약속으로 통신을 할 것인가.

 - OSI 기본 참조 모델,


OSI 기본 참조 모델: 
 - 컴퓨터 통신 기능을 계층 구조로 분할한 개념 모델. (국제표준화기구(ISO)에 의해서 재정)

 - 1~4: 하위계층, 5~7: 상위 계층


Application 계층(L7): 
 - 애플리케이션에 특화된 프로토콜 규정.

Presentation 계층(L6):
 - 데이터 저장 형식, 압축, 문자 코드 등 데이터의 표현 형식을 규정.

Session 계층(L5):
 - 커넥션 확립의 타이밍이나 데이터 전송의 타이밍을 규정.
 - 세션은 애플리케이션 간에 일어난 요청과 응답으로 구성.

Transport 계층(L4):
 - 데이터 전송을 실시.
 - 전송 에러 검출과 재전송을 규정.

 - 통신하는 상대방의 노드에 데이터를 확실하게 보내는 역할.

Network 계층(L3):
 - 다른 네트워크와 통신을 하기 위한 규정.

 - 라우팅: 다른 네트워크에 '데이터 패킷(Packet)'을 전송하는 것.
 - L3의 통신 기기: 패킷을 어디에서 어디로 전송해야 하는지에 대한 정보를 관리.

Datalink 계층(L2):
 - 같은 네트워크 안에 있는 노드 간의 통신을 규정.
 - MAC 주소에 기반하여 데이터를 전송.

 - L2의 통신 기기: 통신하고자 하는 노드가 어떤 포트에 연결되어 있는지를 MAC 주소에 의해서 판단하여 패킷을 전송.

Physical 계층(L1):
 - 통신 기기의 물리적, 전기적 특성을 규정.
 - 데이터를 어떻게 전압과 전류의 값에 할당할 것인가, 혹은 케이블이나 커넥터의 형태 등을 규정.


네트워크를 제어하는 기능을 가진 기기: 
 - 방화벽, 라우터

방화벽:
 - 내부 네트워크와 외부와의 통신을 제어하여, 내부 네트워크의 안전을 유지하기 위한 기술.

 - 유형: 패킷 필터 유형, 애플리케이션 게이트웨이 유형

패킷 필터 유형:
 - 통과하는 패킷을 포트 번호와 IP 주소를 기반으로 필터링하는 방법.
 - ex) "한 IP 주소에서 포트 번호 80번과 443번에게 보낸 패킷만 통과한다", "특정 네트워크에서 온 패킷 이외에는 모두 파기한다" 등의 규칙을 정하고 그 규칙에 근거하여 패킷을 필터링.
 - 패킷 필터링 규칙: 접근제어 리스트(ACL: Access Control List)

애플리케이션 게이트웨이 유형:
 - 패킷이 아닌 애플리케이션 프로토콜로 외부와 통신을 대체하고 제어하는 것. (일반적으로 프록시 서버라고 불림)


라우터:
 - 2개 이상의 다른 네트워크 사이를 중계하기 위한 네트워크 기기. (패킷 중계)
 - OSI 기본 참조 모델의 3계층에서 동작.

 - 경로 선택 기능(라우팅)을 가지고 있음
 - 경로 선택: 어떤 경로를 통하여 데이터를 전송하지를 판단. - 정적 라우팅, 동적 라우팅

 - AWS는 논리적으로 라우팅을 하는 가상 라우팅 기능이 있음.

정적 라우팅:
 - 미리 설정된 라우팅 테이블에 근거하여 결정.
 - 경로를 결정하는 라우팅 테이블을 1대씩 순서대로 관리자가 설정.

동적 라우팅:
 - 라우팅 프로토콜에 근거하여 결정.
 - 라우터 자신이 상호 간에 정보를 서로 교환하여 동적으로 경로를 결정.


L3 스위치:
 - 라우터와 비슷하나,
 - 접속 가능한 이터넷 포트의 수가 많기 때문에 업무 시스템 관련 현장에서 자주 이용.
