단말기에 설치하여 실행하는 애플리케이션을 '네이티브 애플리케이션'이라고 함.
웹 애플리케이션은 네트워크상에 있는 웹 서버에서 프로세싱.
네이티브 애플리케이션은 설치한 단말기상에서 프로세싱 실행.

URL은 웹 애플리케이션의 주소.

서버나 라우터/방화벽 등의 네트워크 기기는 2진수로 프로세싱.

도메인 이름 앞에 호스트 이름을 붙인 것: FQDN(Fully Qualified Domain Name)
FQDN은 IP주소로 변환된 후 통신이 이뤄짐. 이 변환을 담당하는 서버가 DNS 서버.

DNS서버는 계층 구조를 취하고 있음.
인터넷의 최상위 도메인인 루트 도메인을 관리하는 DNS 서버를 루트 서버라고 부름. (트리구조)

DNS서버는  자신이 관리하는 도메인이나 호스트 이름에 해당하는 IP주소를 중복되지 않도록 관리하는 파일을 보유하고 있음.

DNS에서 호스트이름으로부터 IP주소를  물어보는 것: 정방향 DNS 룩업
DNS에서 IP주소로부터 호스트이름을 물어보는 것: 역방향 DNS 룩업

IP주소와 호스트 이름을 대응시키는 것: 도메인 네임 리솔루션(도메인 이름 분석)


*HTTP 프로토콜
 - 클라이언트의 브라우저에서 웹 서버의 콘텐츠나 애플리케이션에 접근하는 프로세싱이 이뤄질 때의 통신 프로토콜.
 - 연결이 이뤄지지 않는 Stateless 통신을 실시.
