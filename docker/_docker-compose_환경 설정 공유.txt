# 1. 여러 개의 컴포즈 파일
 이는 다른 환경이나 워크플로우에서, 컴포즈 애플리케이션을 사용자 지정하는 것을 활성화함.

 기본적으로 `Compose`는 두 개의 파일을 읽는다.
 `docker-compose.yml`, `docker-compose.override.yml`

 `docker-compose.yml`은 기본 구성을 포함하고, `override`파일은 기존 서비스나 완전히 새로운 서비스에 대한 구성 재정의를 포함한다.

 만약 서비스가 두 파일에 모두 정의되었다면, `Adding and overriding configuration`에 설명된 규칙을 사용해서 구성을 병합한다.

 여러 개의 재정의 파일이나 다른 이름의 재정의 파일을 사용하려면 `-f` 옵션을 사용한다.
 
 여러 개의 구성 파일을 사용할 때, 파일에서 모든 경로가 기본 컴포즈 파일에 상대적인지 확인해야한다.
 이는 재정의 파일이 유효한 컴포즈 파일이 아니어도 되기 때문에 필요하다.


 여러 파일에 대한 일반적인 사용 사례는 프로덕젼, 스테이징, CI와 같은 개발 Compose 앱을 변경하는 것임.
 이러한 차이를 지원하기 위해 Compose 구성을 몇 개의 다른 파일로 분할할 수 있음.
