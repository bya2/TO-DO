Compose 확장 서비스:
    - 'Configuration' 추가 혹은 재정의
    - 단일 값 옵션(image, command, mem_limit): 기존 값 대체
    - 다중 값 옵션(ports, expose, external_links, dns, dns_search, tmpfs): 기존 값과 새로운 값 연결
    - (environment, labels, volumes, devices): 새로운 값을 우선순위로 두고 병합
    - (volumes, devices): 컨테이너의 마운트 경로를 사용하여 병합

Compose 내 네트워킹:
    - 앱을 기본적으로 단일 네트워크 설정.
    - 네트워크에 참여한 각 컨테이너는 모두 해당 네트워크의 다른 컨테이너들에 연결될 수 있음.
    - 컨테이너 이름과 동일한 호스트 이름으로 검색할 수 있음.

    - ok 디렉터리에서 docker-compose up 실행
    - 1. ok_default 네트워크 생성.
    - 2. 각 서비스의 'Configuration'을 사용하여 컨테이너 생성 후 ok_default 네트워크에 서비스 이름으로 참여.
    - 3. 2번 반복.
    - 4. 각 컨테이너는 호스트 이름을 검색해서 해당 컨테이너 IP 주소를 가져올 수 있음.
    - 네트워크 서비스 간 통신은 '컨테이너 포트'를 사용함.
    - '호스트 포트'를 정의할 때, 'Swarm' 외부에서도 접근 가능함.

    - 서비스의 구성을 변경하고 `docker-compose up`을 실행한다면 기존 컨테이너는 제거되고, 새로운 컨테이너가 동일한 이름, 다른 IP 주소로 네트워크에 참여함.
    - 어떤 컨테이너가 기존 컨테이너에 대한 연결이 열려있다면, 닫힘. 이 상태를 감지하고 이름을 다시 찾은 후 연결하는 것은 '컨테이너의 책임감'임.

    - `links`는 다른 서비스에서 어떤 서비스에 연결할 수 있는 추가 별칭을 정의할 수 있게함.
