* Swarm 모드

# 용어
** Swarm: 	도커 엔진 클러스터.
** Swarm Mode: 	Swarm 관리(즉, 클러스터 관리). 도커 엔진에 내장.

# 과정
Docker CLI를 사용해서,
1. Swarm 생성,
2. 애플리케이션 서비스를 Swarm에 배포, (즉, Swarm은 애플리케이션 서비스를 배포할 수 있음.)
3. Swarm의 동작 관리.

# Swarm 모드 기능
1. 도커 엔진에 통합된 클러스터 관리: Swarm 생성
2. 분산형 디자인: 단일 디스크 이미지로 전체 Swarm을 빌드할 수 있음.
3. 선언적 서비스 모델: 도커 엔진은 선언적 접근을 사용해서 애플리케이션 스택에서 다양한 서비스의 상태를 원하는대로 정의
4. Scaling: 각 서비스에서, 실행하려는 작업의 수 선언. 규모가 변할 때마다 Swarm 관리자는 원하는 상태를 유지하기 위해 자동적으로 작업을 추가하거나 제거하여 적용.
5. 원하는 상태 조정: Swarm 관리자 노드가 지속적으로 클러스터 상태 모니터링 및 실제 상태와 원하는 상태 간에 차이점을 조정.
6. 다중 호스트 네트워킹: 서비스에서 오버레이 네트워크 지정. Swarm 관리자는 오버레이 네트워크에서 컨테이너에 주소 할당. (그것이 애플리케이션을 초기화하고 업데이트했을 때)	
7. 서비스 검색: Swarm 관리자 노드가 각 서비스에 고유 DNS 이름을 할당, 실행 중인 컨테이너의 로드 밸런스 조정. Swarm에 내장된 DNS 서버를 통해 Swarm에서 실행 중인 모든 컨테이너를 쿼리할 수 있음.
8. 로드 밸런싱: 외부 로드 밸런서에 서비스에 대한 포트를 표시. 내부적으로 Swarm은 노드 간에 서비스 컨테이너를 분배하는 방법을 사용자가 지정하게 함.
9. 기본적인 보안: Swarm의 각 노드는 노드 간 통신 보안을 위해 TLS mutual authentication(TSL 상호 인증)과 encryption(암호화)을 강요.
10. Rolling 업데이트: 원격 설치 시, 서비스 업데이트를 노드에 점진적으로 적용. Swarm 관리자는 서로 다른 노드 집합에 대한 서비스 배포 간에 사용자가 지연을 제어하도록함.
