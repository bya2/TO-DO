@Docker Architecture
Docker Client: (docker)
    - 도커 유저들이 도커와 상호작용(커맨드)하는 주요 방법. (클라이언트는 커맨드를 데몬에 보냄)
    - 여러 개의 데몬과 통신 가능.
    - 도커 클라이언트는 컨테이너 집합으로 구성된 애플리케이션으로 작업하는 '도커 Compose'임.

Docker Daemon: (dockerd)
    - 역할: 도커 API 요청 수신, 도커 객체 관리
    - 도커 데몬은 도커 서비스를 관리하는 다른 도커 데몬과 통신할 수 있음.

Docker Client - Docker Deamon
    - 클라이언트-서버 아키텍처.
    - 도커 클라이언트와 도커 데몬은 같은 시스템에서 실행될 수 있고, 도커 클라이언트를 원격 도커 데몬에 연결할 수 있음.
    - 도커 클라이언트와 도커 데몬은 UNIX 소켓 혹은 네트워크 인터페이스를 통해 REST API를 사용하여 통신함.
