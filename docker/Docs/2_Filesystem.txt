Container's filesystem:
    - 컨테이너를 실행하면, 그 파일 시스템을 위해 이미지에서 다양한 레이어 사용.
    - 각 컨테이너에는 파일들을 생성/갱신/제거 하기 위한 자체 'Scratch space'가 존재.


컨테이너들이 파일들을 생성/갱신/제거할 수 있는 반면, 그 변경 사항들은 컨테이너가 제거됬을 때 사라지고, 모든 변경사항들은 컨테이너에 고립되어있음.

볼륨을 통해서 이 모든 것을 해결할 수 있음.

볼륨:
    컨테이너의 특정 파일 시스템 경로를 호스트 시스템에 연결하는 기능을 제공.
    컨테이너 내 디렉터리가 마운트된다면, 해당 디렉터리 내 변경 사항들은 호스트 시스템에도 표시됌.
    컨테이너를 재시작을 통해 동일한 디렉터리를 마운트하면, 동일한 파일들이 확인됌.

    볼륨을 생성하고, 데이터가 저장될 디렉터리에 볼륨을 첨부함으로써, 데이터를 유지할 수 있음.

    -도커는 디스크에서 물리적 위치를 유지하고, 사용자는 볼륨의 이름만 기억하면 됌.
    볼륨을 사용할 때마다, 도커는 올바른 데이터가 제공되었는지 확인할 것임.

    - 볼륨 생성:
	- docker volume create <volume-name>

    - 볼륨 플래그:
	- '-v <volume-name>:<directory>

    - 볼륨을 사용할 때, 도커가 데이터를 저장하는 실제 위치:
	- docker volume inspect <volume-name>
	- mountpoint
	- 대부분의 컴퓨터에서는 호스트에서 이 디렉터리에 접근하려면 루트 접근 권한을 필요로 함.

    *?? 도커 데스크톱에서 실행하는 동안 도커 명령은 실제로 시스템의 작은 VM 내에서 실행됩니다. Mountpoint 디렉토리의 실제 콘텐츠를 보려면 먼저 VM 내부로 들어가야 합니다.
