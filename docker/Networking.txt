* Networking

 도커 네트워킹 하위시스템은 '드라이버'를 사용하여 플러그가 가능.
 몇몇 드라이버는 기본적으로 존재하고, 핵심 네트워킹 기능을 제공.

 - bridge	: `standalone` 컨테이너가 통신할 때 사용하는 드라이버가 bridge인 네트워크
    : 디폴트. 
    : 통신해야하는 standalone 컨테이너에서 실행될 때 사용.
    : 동일한 도커 호스트에서 통신에 다중 컨테이너가 필요할 때 가장 좋음. (User-defined bridge network)

 - host	: 
    : standalone 컨테이너에서 컨테이너와 도커 호스트 간에 네트워크 분리 제거. 그 다음 호스트 네트워킹을 직접 사용.
    : 네트워크 스택을 도커 호스트에서 분리하면 안되지만, 컨테이너의 다른 측면을 분리하려는 경우 가장 좋음.

 - overlay	: 
    : 다중 도커 데몬을 함께 연결, swarm 서비스 활성화. (서로 간에 통신하기 위해)
    : swarm 서비스와 standalone 컨테이너 간의 통신, 다른 도커 데몬에서 두 standalone 컨테이너 간의 통신을 용이하게 할 수 있음.
    : 컨테이너 간에 OS-level 라우팅을 할 필요 없음.
    : 다른 도커 호스트에서 통신을 위해 실행되는 컨테이너가 필요할 때, Swarm 서비스를 사용하여 다중 애플리케이션을 함께 작업할 때 가장 좋음.
